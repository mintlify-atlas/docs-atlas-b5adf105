## What is Tooltify?

Tooltify bridges the gap between your running React app and your source code. It injects a lightweight devtools panel directly into your browser during development, allowing you to:

- **Inspect any element** and jump to its source file instantly
- **Edit source files** in a Monaco editor without leaving the browser
- **Track builds** in real time — see which user triggered each rebuild and whether it succeeded
- **Open files in your IDE** via a WebSocket-connected local agent

## How it works

<Steps>
  <Step title="Start the local server">
    The `@tooltify/core` server runs alongside your dev server, exposing an HTTP API and WebSocket namespaces for agent communication and build events.
  </Step>
  <Step title="Inject the client bundle">
    The bundler plugin (e.g. `rspackTooltify()`) embeds the Tooltify client script into your HTML and replaces React's JSX dev runtime with one that tags every DOM element with its source location.
  </Step>
  <Step title="Connect the desktop agent">
    Run `tooltify` from your terminal to start the local agent daemon. It connects back to the server over WebSockets and executes IDE-open commands on your machine.
  </Step>
  <Step title="Develop faster">
    Click any element in the browser, see its source path, edit it inline, and watch the build tracker report who changed what — all without switching windows.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="In-browser source editor" icon="file-pen">
    Monaco-powered editor embedded in the devtools panel. Read and save source files directly from the browser.
  </Card>
  <Card title="Build tracking" icon="chart-line">
    Tracks every rebuild, attributes file changes to the author who saved them, and streams status events to all connected clients.
  </Card>
  <Card title="Open in IDE" icon="arrow-up-right-from-square">
    Cross-platform agent supports VS Code, Cursor, and other editors on Linux, macOS, and Windows.
  </Card>
  <Card title="JWT authentication" icon="lock">
    Session-based JWT auth protects the HTTP API and WebSocket connections so the devtools panel is only accessible to registered team members.
  </Card>
  <Card title="Rspack integration" icon="bolt">
    First-class Rspack plugin — zero manual wiring. The plugin handles server startup, HTML injection, and build event forwarding automatically.
  </Card>
  <Card title="Source injection" icon="code">
    Replaces React's JSX dev runtime to attach `data-source` attributes on every rendered DOM element, enabling precise source-to-element mapping.
  </Card>
</CardGroup>
