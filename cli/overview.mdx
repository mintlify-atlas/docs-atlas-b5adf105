The `tooltify` CLI lets you start, stop, and inspect the Tooltify desktop agent daemon that runs on your local machine. It is an interactive, menu-driven tool — you run it once, pick an action, and it walks you through any required inputs.

## Installation

The CLI ships with `@tooltify/core`. Install it globally or use `npx`:

```bash
# Global install
npm install -g @tooltify/core

# Or run without installing
npx @tooltify/core
```

After a global install, the binary is available as `tooltify`.

## Prerequisites

<Warning>
  You must run `tooltify` from a directory that contains a `tooltify.config.json` file. If the file is not found, the CLI exits with an error before showing the menu.
</Warning>

The CLI reads project configuration (including the auth salt and registered users) from `tooltify.config.json` in your working directory. Make sure you `cd` into the project root before running any command.

## How it works

When you run `tooltify`, it:

<Steps>
  <Step title="Verify the project">
    Checks for `tooltify.config.json` in the current directory.
  </Step>
  <Step title="Show the interactive menu">
    Prompts you to select an action. Your OS username (from `os.userInfo().username`) is used as the agent name and shown next to relevant menu items.
  </Step>
  <Step title="Run the selected command">
    Executes the action — some commands prompt for additional input before proceeding.
  </Step>
</Steps>

### Example session

```bash
$ tooltify
◆  Tooltify Agent Manager
│
◆  What do you want to do?
│  ● Start agent (alice)
│    Stop agent (alice)
│    Reset password (alice)
│    View status
│    View logs
└
```

Press `Ctrl+C` at any prompt to cancel the current operation and exit cleanly.
