---
title: "Commands"
description: "Detailed reference for every action available in the tooltify interactive CLI."
---

All commands are accessed through the `tooltify` interactive menu. Run `tooltify` from your project root (the directory containing `tooltify.config.json`) and select the action you want.

<Note>
  The **agent name** for every command is your OS username, resolved at runtime via `os.userInfo().username`.
</Note>

<AccordionGroup>

  <Accordion title="start" id="start">

    Starts the local agent daemon. On the first run, it also registers your credentials before launching the agent.

    ### Flow

    <Steps>
      <Step title="Credential check">
        The CLI reads `~/.tooltify/credentials.json` (Linux/macOS) or `%APPDATA%\tooltify\credentials.json` (Windows). If no credentials file exists, it prompts you to create a password before continuing.

        ```
        Enter password  ****
        Confirm password  ****
        ```

        Passwords must match. If they do not, the CLI logs an error and exits without starting the agent. Once accepted, the password hash is written to the credentials file and recorded in `tooltify.config.json` under your username.
      </Step>
      <Step title="Editor selection">
        Choose the IDE that Tooltify should integrate with:

        ```
        ◆  Editor
        │  ● VS Code
        │    Antigravity
        └
        ```
      </Step>
      <Step title="Remote mode">
        Specify whether the editor is running over a remote connection (VS Code Remote WSL or SSH):

        ```
        ◆  Is the editor running remotely? (e.g. VS Code Remote WSL/SSH)
        │  ● Yes — remote (WSL / SSH)
        │    No — local machine
        └
        ```
      </Step>
      <Step title="Daemon launch">
        The CLI spawns the agent as a detached background process. A spinner shows progress, then reports the PID on success:

        ```
        ◆  Starting agent (alice)...
        ◆  Agent started (alice) — PID: 12345
        ```

        The daemon's stdout and stderr are written to the agent log file (see [logs](#logs)).
      </Step>
    </Steps>

    ### Credentials file location

    | Platform     | Path                                         |
    | ------------ | -------------------------------------------- |
    | Linux / macOS | `~/.tooltify/credentials.json`              |
    | Windows      | `%APPDATA%\tooltify\credentials.json`        |

    The file is written with mode `0o600` (owner read/write only).

    ### Settings persistence

    After a successful start, the daemon saves its configuration to an agent settings file in the platform agent directory (`/tmp/tooltify/agents/<username>.settings.json` on Linux/macOS). This file records `ideType`, `remote`, and the working directory, and is used by `reset-password` to restart the daemon with the same settings.

    <Tip>
      If the agent is already running when you select **start**, the existing process is terminated before the new one is spawned.
    </Tip>

  </Accordion>

  <Accordion title="stop" id="stop">

    Stops the running agent daemon for your user.

    The CLI looks up the PID from the agent PID file. If no daemon is running, it logs a warning and exits:

    ```
    ▲  Agent "alice" is not running
    ```

    If the agent is running, a spinner shows progress and reports the stopped PID:

    ```
    ◆  Stopping agent (alice)...
    ◆  Agent stopped (alice) — PID: 12345
    ```

    The PID file is removed after the process is killed.

    <Warning>
      Stop sends a termination signal directly to the process. Make sure any in-progress agent work completes before stopping if your workflow requires it.
    </Warning>

  </Accordion>

  <Accordion title="status" id="status">

    Displays the current state of all registered agents.

    The CLI reads every `.pid` file in the agent directory and checks whether the recorded process is still alive. Output is one line per agent:

    ```
    ●  alice   PID: 12345   running
    ○  bob     PID: 99001   stopped
    ```

    - `●` — the process is alive and responding to signal 0.
    - `○` — the PID file exists but the process is no longer running.

    If no PID files are found, the CLI warns that there are no registered agents:

    ```
    ▲  No registered agents
    ```

  </Accordion>

  <Accordion title="logs" id="logs">

    Streams live log output from the agent daemon.

    <Steps>
      <Step title="Agent selection">
        If more than one agent is registered, you are prompted to pick one from the list. If no agents are registered, the CLI exits with a warning.
      </Step>
      <Step title="Log snapshot">
        The full current contents of the log file are printed immediately so you have context before live output begins.
      </Step>
      <Step title="Live tail">
        New content is streamed to stdout as the daemon writes it. The log file is polled every 500 ms for new bytes.

        ```
        Tailing /tmp/tooltify/agents/alice.log  (Ctrl+C to exit)

        [2026-03-18T10:00:01Z] Agent started
        [2026-03-18T10:00:02Z] Listening on port 4242
        ...
        ```

        Press `Ctrl+C` to stop tailing and exit.
      </Step>
    </Steps>

    ### Log file location

    | Platform     | Path                                              |
    | ------------ | ------------------------------------------------- |
    | Linux / macOS | `/tmp/tooltify/agents/<username>.log`            |
    | Windows      | `%ProgramData%\tooltify\agents\<username>.log`   |

    <Note>
      The log command tails the file in place — it does not follow log rotation. If the daemon is restarted and a new log file is created, restart the `logs` command to tail the new file.
    </Note>

  </Accordion>

  <Accordion title="reset-password" id="reset-password">

    Changes the agent user's password and, if the daemon is currently running, restarts it automatically with the new credentials.

    <Steps>
      <Step title="Enter new password">
        You are prompted to enter and confirm the new password:

        ```
        Enter new password  ****
        Confirm new password  ****
        ```

        If the two values do not match, the CLI logs an error and exits without making any changes.
      </Step>
      <Step title="Credentials update">
        The password is hashed with SHA-256 (salted from `tooltify.config.json`) and written to the credentials file. The hash is also updated in `tooltify.config.json` under your username.

        ```
        ✓  Password reset for "alice"
        ```
      </Step>
      <Step title="Daemon restart (if running)">
        If the agent daemon is currently running, the CLI reads the saved settings, stops the current process, and starts a new one with the updated hash:

        ```
        ◆  Restarting agent (alice)...
        ◆  Agent restarted (alice) — PID: 12346
        ```

        If the daemon is not running, no restart is attempted.
      </Step>
    </Steps>

    <Warning>
      Resetting the password updates `tooltify.config.json` in your project directory. Commit the updated file so that other users and CI environments pick up the change.
    </Warning>

  </Accordion>

</AccordionGroup>
